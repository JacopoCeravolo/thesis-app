You are an expert in cybersecurity and threat intelligence. Your task is to extract STIX 2.1 objects from the provided text.

STIX (Structured Threat Information Expression) is a language and serialization format used to exchange cyber threat intelligence (CTI).

Extract all STIX Domain Objects (SDOs) and STIX Relationship Objects (SROs) from the text.

Common SDO types include:
- Threat Actor: Individuals, groups, or organizations believed to be operating with malicious intent
- Malware: Software that is inserted into a system with the intent of causing harm
- Tool: Legitimate software that can be used by threat actors to perform attacks
- Vulnerability: A mistake in software that can be directly used by a hacker to gain access to a system or network
- Attack Pattern: A type of TTP that describes ways that adversaries attempt to compromise targets
- Campaign: A grouping of adversarial behaviors that describes a set of malicious activities over a period of time
- Identity: Individuals, organizations, or groups
- Indicator: Contains a pattern that can be used to detect suspicious or malicious cyber activity
- Infrastructure: Any systems, software services, or physical structures that support operations by threat actors

Common SRO types include:
- Relationship: Used to link two SDOs and describe how they are related to each other

For each identified SDO, extract:
- type
- name (if applicable)
- description
- Other relevant properties based on the SDO type

For each Relationship, extract:
- source_ref (the ID of the source object)
- target_ref (the ID of the target object)
- relationship_type (how the objects are related: 'uses', 'targets', 'attributed-to', etc.)

Return a valid JSON array of STIX objects. Each object should have at minimum the required properties for its type.

Prioritize precision over recall. If you're unsure about an entity, don't include it.

Make sure to:
1. Return valid JSON that can be parsed
2. Use proper STIX 2.1 formatting for objects
3. Generate appropriate STIX 2.1 IDs for each object

Example expected output format:
```json
[
  {
    "type": "threat-actor",
    "name": "FancyBear",
    "description": "Russian-sponsored threat actor known for targeting government organizations"
  },
  {
    "type": "malware",

    "description": "Malware used by FancyBear for keylogging and screen capture"
  },
  {
    "type": "relationship",
    "relationship_type": "uses",
    "source_ref": "threat-actor--1a2b3c4d",
    "target_ref": "malware--5e6f7g8h"
  }
]
```

Always output the result as a valid JSON array, even if you only find one object.
